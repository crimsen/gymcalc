Einrichten eines Wettkampfes

Ein Wettkampf wird in einer Datei mit der Endung 'contest' verwaltet. In dieser Datei sind alle Aktiven, Kampfrichter, Wettkampfklassen, Riegen und weitere enthalten. Um eine solche Datei neu einzurichten sind folgende Schritte notwendig:

  • Starten von GymCalc und Anlegen eines Arbeitsplatzes
  • Vorbereiten des Arbeitsplatzes
  • Anlegen eines Projektes
  • Anlegen der Wettkampfdatei
  • Eingabe der Bezeichung und des Datums
  • Anlegen von Geräten bzw. Stationen
  • Anlegen von Wettkampfklassen
  • Anlegen von Riegen

Procedure 3. Starten von GymCalc und Anlegen eines Arbeitsplatzes

In der Regel wurde diese Prozedur schon während der Installation ausgeführt. Will allerdings ein anderer Nutzer als der, der GymCalc installiert hat, GymCalc nutzen, so ist diese Prozedur notwendig.

  1. GymCalc wird durch Doppelklick auf Eclipse im Installationsverzeichis gestartet da es als "Plugin" d.h. als Zubehör des viel mächtigeren Programmes Eclipse entwickelt worden ist.

  2. Klicke OK im Select Workspace-Dialog um das Defaultverzeichis für den Arbeitsplatz zu verwenden.

    In diesem Verzeichnis werden Arbeitsplatz-spezifische Daten wie Projekte, Fensteranordnung, Dialogeinstellungen usw. gespeichert.

  3. Klicke Workspace im nun einmalig erscheinenden initialen Fenster um zur eigentlichen Arbeitsfläche zu kommen.

  4. Fahre mit Vorbereiten des Arbeitsplatzes fort.

Procedure 4. Vorbereiten des Arbeitsplatzes

Nachdem GymCalc erstmalig gestartet wurde, erscheint ein Fenster mit geöffneten Teilfenstern bzw. "Views" wie sie zum Entwickeln neuer Programme häufig verwendet werden. Diese Ansicht ist für GymCalc unpraktisch und sollte wie folgt geändert werden.

Öffnen des "Navigator"-View.
Öffnen des "Outline"-View.
Schliessen aller anderen Views.

  1. Wähle Window → Show View → Navigator

  2. Schließe alle Views außer Navigator und Outline durch Drücken auf . in den Views.

  3. Verschiebe Navigator und Outline entsprechend dem Screenshot.

  4. Fahre mit Anlegen eines Projektes fort.

Procedure 5. Anlegen eines Projektes

Zur Benutzung von GymCalc ist es notwendig ein Projekt anzulegen. Das Projekt ist in diesem Fall eine Referenzierung auf ein Verzeichnis auf der Festplatte in dem alle zum Wettkampf gehörenden Dateien wie Ausschreibung, eingehende Meldungen usw. abgelegt werden können.

  1. Wähle File → New → Project...

  2. Im erscheinenden Dialog wähle General → Project und wähle Next >.

  3. Hier ist ein Projektname einzusetzen beispielsweise 'gymcalc'. Es kann natürlich auch ein anderer Name gewählt werden.

Procedure 6. Anlegen der Wettkampfdatei

  1. Wähle File → New → Other...

  2. Im erscheinenden Dialog wähle Example EMF Model Creation Wizards → Contest Model und wähle Next >.

  3. Wähle das Projekt und das Verzeichnis in dem die Datei erstellt werden soll, gib unter File name: einen Dateinamen mit der Endung ".contest" an und wähle Next >.

  4. In der erscheinenden Seite ist das Model Object "Contest" auszuwählen und der Dialog mittels Finish zu beenden.

    Damit wird die Wettkampfdatei im zuvor ausgewähltem Projekt und Verzeichnis angelegt und die Übersicht des Wettkampfes geöffnet. Folgende Schritte sollten nun sein den Wettkampf mit entsprechenden Parametern zu versehen.

Procedure 7. Eingabe der Bezeichung und des Datums

Bezeichnung und Datum des Wettkampfes werden mittels "Übersicht"-Editor eingegeben. Dieser Editor bzw. dieses Fenster, wird nach Anlegen einer Wettkampfdatei automatisch geöffnet. Manuell kann er mittels Rechts Klick auf Wettkampfdatei im Navigator Open With → Übersicht geöffnet werden.

  1. Eingabe der Wettkampfid: "Id".

    Die "Id" ist eine Bezeichnung, die den Wettkampf eindeutig identifiziert der Form: '_<Datum>-<wettkampfname>' Beispiel '_2014-03-01-reife-jahresringe'. Darf keine Leerzeichen enthalten und muss mit einem Unterstrich oder Buchstaben beginnen. Die ID wird nur für maschinelle Weiterverarbeitung der Daten verwendet, beispielsweise wenn mehrere Wettkämpfe verglichen werden sollen.

  2. Eingabe des Wettkampfnames: "Name".

    Name des Wettkampfes wie er in Ausschreibungen und Protokollen erscheint. Beispielsweise :' XIX. Turnen der Reifen Jahresringe Spremberg'

  3. Eingabe des Wettkampfdatums "Date" erfolgt durch drücken auf .

    Datum, an dem der Wettkampf stattfindet. Bei mehrtägigen Wettkämpfen ist der erste Tag anzugeben. Die angezeigte Uhrzeit ist nicht notwendig und programmatisch bedingt.

    Note

    Die Darstellung erscheint englisch in der Form Monat/Tag/Jahr. Diese Darstellung ebenso wie die Anzeige der Uhrzeit sind programmatisch bedingt und werden in kommenden Versionen berichtigt.

  4. Speichere die Datei mittels File → Save

Procedure 8. Anlegen von Geräten bzw. Stationen

Geräte bzw. Stationen sind wichtig, um einen Wettkampf mit riegenweisem Wechsel zwischen Geräten zu organisieren. Ausserdem werden an diesen Stationen Kampfrichter eingesetzt. Bei mehrkreisigen Wettkämpfen sollten die Stationen sich im Namen unterscheiden: 'Balken 1', 'Balken 2'. Auch mehrfach genutzter Boden sollte sich unterscheiden: 'Boden 1', 'Boden 2' da jeder Station höchstens ein Kampfgericht zugeordnet werden kann.

Note

Folgende Schritte beziehen sich auf den geöffneten "Übersicht"-Editor. Ist dieser noch nicht offen, so kann er wie bei Eingabe der Bezeichnung geöffnet werden.

  1. Im Bereich Geräte, Stationen klicke auf . Das Öffnen dauert leider einige Sekunden. Nach diesen ... Sekunden ... ist der New Reference Element-Dialog geöffnet. ... beim 2. Mal geht das deutlich schneller.

  2. Wähle die einzige Auswahl StationType und drücke Finish. Es erscheint wiederum ein Dialog zur Eingabe der Informationen bezüglich einer Station.

  3. Gib im Feld Id einen eindeutigen Bezeichner für diese Station ein. Dieser Bezeichner wird nur zum Referenzieren verwendet. Er darf keine Leerzeichen enthalten und muss mit einem Buchstaben oder einem Unterstrich beginnen. Bei mir hat sich eine Bezeichnung 'station-'<stationname> durchgesetzt. Also beispielsweise: 'station-sprung. Künftig wird GymCalc diese Id's verstecken und selbst generieren.

  4. Im Feld Name ist der Name der Station einzugeben.

  5. Drücke OK.

  6. Diese Schritte sind für alle Geräte bzw. Stationen des Wettkampfes zu wiederholen. Sind im Wettkampf Pausengeräte, dann sollten auch diese eingetragen werden. Dies ist sinnvoll, da die Abfolge der Stationen nach der Zuordnung zu Riegen, in den Riegenlisten ausgegeben wird.

  7. Speichere die Datei mittels File → Save

Procedure 9.  Anlegen von Wettkampfklassen

Eine Wettkampfklasse ist eine Einteilung für die es eine Plazierungsreihenfolge gibt bzw. für die man eine Siegerehrung durchführen kann. Beispiele sind: Einzelwertung im weiblichen Bereich Jugend LK4, Mannschaftswertung im weiblichen Bereich Jugend LK4 oder Finale Boden weiblich Jugend LK4. Finals werden in der Regel aus Einzelwertungen generiert. Hier soll das Anlegen beispielhaft für die Einzelwertungen in einem Wettkampf gezeigt werden.

Note

Folgende Schritte beziehen sich auf den geöffneten "Übersicht"-Editor. Ist dieser noch nicht offen, so kann er wie bei Eingabe der Bezeichnung geöffnet werden.

  1. Im Bereich Wettkampfklassen klicke auf . Das Öffnen dauert leider einige Sekunden. Nach diesen ... Sekunden ... ist der New Reference Element-Dialog geöffnet. ... beim 2. Mal geht das deutlich schneller.

  2. In diesem Dialog ist die einzige Auswahl ClassType zu selektieren und auf Finish zu klicken. Es öffnet sich ein weiterer Dialog zur Eingabe der Parameter einer Wettkampfklasse.

  3. Gib den Namen der Wettkampfklasse im Feld Nameein. Der Name sollte sich eindeutig von anderen Wettkampfklassen unterscheiden und er sollte verständlich sein, da dieser Name später auch in Protokollen und anderen Wettkampflisten verwendet wird. Bei mir haben sich kurze prägnante Namen wie 'wJLK4' für weiblich Jugend LK4 durchgesetzt.

  4. Wenn es sich um eine spezielle Berechnung handelt, dann ist diese Berechnungsmethode im Feld Calculationkey einzugeben. Ist hier nichts eingetragen, so werden die einzelnen Wertungen einfach addiert. Eine spezielle Berechnungsmethode ist beispielsweise wenn nur die besten 3 Wertungen zusammenrechnen werden sollen. Welche Berechnungsmethoden möglich sind wird in der Referenz zum "Calculationkey" angegeben.

  5. Mit werden einzelne Disziplinen zu einer Wettkampfklasse hinzugefügt.

    1. Wähle die einzige Auswahl DisziplineType und drücke Finish. Es erscheint wiederum ein Dialog zur Eingabe der Informationen bezüglich einer Wettkampfdisziplin.

    2. Gib im Feld Id einen eindeutigen Bezeichner für diese Disziplin ein. Dieser Bezeichner wird nur zum Referenzieren verwendet. Er darf keine Leerzeichen enthalten und muss mit einem Buchstaben oder einem Unterstrich beginnen. Bei mir hat sich eine Bezeichnung <Wettkampfklasse>-<Disziplinname> durchgesetzt. Also beispielsweise: 'wjlk4-boden'. Künftig wird GymCalc diese Id's verstecken und selbst generieren.

    3. Im Feld Name ist der Name der Disziplin einzugeben. Für das Gerätturnen sollten das Sprung, Stuba, Balken, Boden, Pferd, Ringe, Barren oder Reck sein.

    4. Drücke auf um der Disziplin ein Gerät bzw. eine Station zuzuordnen. Es wird angenommen, dass jede Disziplin an genau einer Station geturnt wird und von einem Kampfgericht bewertet wird.

      Hat man noch keine Stationen angelegt, dann kann muss die Zuordnung nachgeholt werden. Siehe Anlegen einer Station

      1. Wähle die Station aus, an der die Disziplin geturnt wird und drücke Finish.

      2. Drücke OK. Weder Id noch Name der Station bzw. des Gerätes sollen hier geändert werden.

    5. Man kann noch keine Lookuptable in diesem Dialog hinzufügen. Lookuptable sind für spezielle Berechnungen notwendig. Spezielle Berechnungen wenn z.B. Zeiten beim Sprint in Punkte gemäß einer Tabelle umgerechnet werden sollen.

    6. Calculationkey ist wiederum für besondere Berechnungsmethoden einer Disziplin notwendig. Beispiel hier ist 'force', wenn diese Disziplin bei einer Berechnungsmethode nur die besten 3 Wertungen auf alle Fälle mit dabei sein muss. Detailierte Informationen finden sich in der Referenz.

    7. Drücke auf OK.

    8. Wiederhole die vorigen Schritte zum Anlegen einer Disziplin, bis alle Disziplinen einer Wettkampfklasse hinzugefügt wurden.

  6. Drücke OK, wenn alle Disziplinen hinzugefügt wurden.

  7. Wiederhole die vorigen Schritte zum Anlegen einer Wettkampfklasse, bis alle Wettkampfklassen hinzugefügt wurden.

  8. Speichere die Datei mittels File → Save

Procedure 10. Anlegen von Riegen

Note

Folgende Schritte beziehen sich auf den geöffneten "Übersicht"-Editor. Ist dieser noch nicht offen, so kann er wie bei Eingabe der Bezeichnung geöffnet werden.

  1. Im Bereich Riegen klicke auf . Das Öffnen dauert leider einige Sekunden. Nach diesen ... Sekunden ... ist der New Reference Element-Dialog geöffnet. ... beim 2. Mal geht das deutlich schneller.

  2. Wähle die einzige Auswahl ChainType und drücke Finish. Es erscheint wiederum ein Dialog zur Eingabe der Informationen bezüglich einer Riege.

  3. Im Feld Name ist der Name der Riege einzugeben. Gewöhnlich sind die Riegen einfach durchnummeriert in der Art: Riege 1, Riege 2 ...

  4. Im Listenfeld Station ist die Geräteabfolge der Riege einzugeben.

    1. Drücke auf .

    2. Wähle die erste Station, bzw. das erste Gerät für diese Riege aus, drücke Finish.

    3. Drücke OK. Weder Id noch Name der Station bzw. des Gerätes sollen hier geändert werden.

    4. Füge so alle Stationen für diese Riege ein. Beachte, dass die Stationen in der richtigen Reihenfolge referenziert werden. Leider ist ein Umsortieren mit dem "Übersicht"-Editor leider noch nicht möglich.

  5. Drücke OK.

  6. Diese Schritte sind für alle Geräte bzw. Stationen des Wettkampfes zu wiederholen. Sind im Wettkampf Pausengeräte, dann sollten auch diese eingetragen werden. Dies ist sinnvoll, da die Abfolge der Stationen nach der Zuordnung zu Riegen, in den Riegenlisten ausgegeben wird.

  7. Speichere die Datei mittels File → Save

Das Resultat sollte nun ein fertig angelegter Wettkampf mit einer Wettkampfklasse sein. Weitere Wettkampfklassen sind analog zu Anlegen von Wettkampfklassen anzulegen.
